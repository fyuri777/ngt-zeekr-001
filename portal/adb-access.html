ADB-доступ к Zeekr 001 — что можно, что нельзя, риски

Обзор практики владельцев Zeekr 001 за август 2024 — май 2026: что говорят 2755+ сообщений о возможностях ADB, подводных камнях обновлений и реальной цене ошибок.

Как получить ADB-доступ

Головное устройство (DHU) Zeekr 001 работает на Android, и ADB (Android Debug Bridge) — основной инструмент для установки приложений, настройки системы и диагностики. Доступ открывается через инженерное меню, но способ зависит от поколения автомобиля и версии прошивки.

Дорестайл и прошивки до 6.0.8. ADB можно открыть самостоятельно или удалённо через специалиста с Windows-компьютером и USB-кабелем. Стоимость услуги — 5–15 тыс. руб. (Info Hub, русификаторы). Инженерное меню вызывается командой adb shell am start -n com.yfvet.engineeringmode/.main.MainActivity или физически — одновременным зажатием обеих крутилок на руле до перезагрузки. 👥 CONSENSUS

Рестайл и прошивки 6.1+. Начиная с 6.1 на инженерном меню появился динамический пароль. Самостоятельное открытие без специальных инструментов невозможно — пароль лечится подменой загрузчика. Стоимость открытия у специалистов: 10–60 тыс. руб., для рестайлов 2024–2025 годов — до 150–200$. На 6.3.1+ штатная инженерка перестаёт открывать ADB — нужна альтернативная версия APK от прошивки 6.0.8. 🆕 RECENT

Альтернативный путь — установить приложение zSupport, которое переключает режим USB-порта без пароля инженерки. Также возможно открытие через OBD-диагностику (оборудование ZDS + ноутбук).

Что нужно для подключения

Для работы с ADB потребуется компьютер (Windows предпочтительнее), USB-кабель и установленный ADB-драйвер. Несколько критичных нюансов:

Кабель: строго USB Type-A → Type-C. Кабели Type-C → Type-C часто не работают. Кабель должен поддерживать передачу данных (data), а не только зарядку. Рекомендуются короткие толстые кабели от SSD (Samsung и аналогичные). 👥 CONSENSUS

Порт в машине: используется левый USB-порт в подлокотнике. Правый — только питание (зарядка 27W, до 120W с переходником).

Компьютер: MacBook на M-чипах может не видеть машину из-за отсутствия драйверов или особенностей Type-C. Windows 11 Home тоже иногда вызывает проблемы. Помогает смена USB-порта на ноутбуке или другой компьютер. ⚠️ STALE

Режим порта: для ADB порт должен быть в режиме Peripheral (Device). Для флешки — в режиме Host. Переключается через zSupport или инженерное меню. Флешка читается только в FAT32, exFAT не поддерживается.

Что можно делать через ADB

ADB на Zeekr 001 даёт доступ уровня обычного пользователя Android — без root, но с широкими возможностями. Архитектура процессора — ARM64, при скачивании APK нужно выбирать соответствующую версию.

Установка приложений. Яндекс Навигатор, Яндекс Музыка, RuStore, ZeeApp Store, Telegram, VPN-клиенты (Amnezia VPN), Google Keyboard, Apple Music (старые версии), APKPure, FX Explorer. Установка через adb install -g <файл.apk>. 👥 CONSENSUS

Выдача разрешений. VPN для DNS Changer: adb shell appops set --user 0 com.andevstudioth.changedns ACTIVATE_VPN allow. Права на установку пакетов для Telegram: adb shell appops set org.telegram.messenger.web REQUEST_INSTALL_PACKAGES allow. Аналогично для APKPure: пакет com.apkpure.aegon.

Отключение верификатора пакетов (обязательно для установки сторонних APK на 6.3+):
- adb shell pm disable-user --user 0 com.ecarx.xsfinstallverifier
- adb shell settings put global package_verifier_enable 0
- adb shell settings put global verifier_verify_adb_installs 0
- adb reboot

На 6.1+ эти команды не работают без открытой инженерки. После перезагрузки настройки могут сброситься.

Системные настройки. Смена часового пояса: adb shell service call alarm 3 s16 Europe/Moscow (не слетает при перезагрузке, но приборка в 99% случаев не обновится). Очистка кэша приложения: adb shell pm clear ru.yandex.yandexnavi. Список сторонних приложений: adb shell pm list packages -3. Вызов меню обновления: adb shell am start -n ecarx.upgrade/ecarx.upgrade.ui.activity.Dc1eMainActivity.

Извлечение пароля от бардачка. Требуется временный псевдо-root: adb shell setprop service.adb.root 1, затем adb shell 'cat /data/data/com.geely.pma.settings/shared_prefs/SpUtil.xml | grep def_settings_glove_box_value'. Работает только на дорестайлах.

Удаление русификации при глюках: adb uninstall com.geely.pma.settings (настройки) или adb uninstall com.zeekr.carlauncher3d (лаунчер). Часто единственный способ восстановления при зависании на заставке.

Чего нельзя делать — ограничения root

Полноценный root на Zeekr 001 закрыт. На дорестайлах был временный псевдо-root через adb shell setprop service.adb.root 1 — работает до перезагрузки, только для чтения системных файлов. На рестайлах с прошивки 6.0.2+ root полностью закрыт. 👥 CONSENSUS

Невозможно: установить Magisk/su, модифицировать системные разделы, прошивать кастомные ROM, получить постоянный root. Для постоянного root требуется патченная прошивка DHU — это крайне рискованно и может превратить головной блок в кирпич.

zSupport — ключевое приложение

zSupport (версия 1.3.3+ / 1.3.5+) — приложение для управления ADB, языком, часовым поясом и режимом USB-порта. Доступно в RuStore и ZeeApp Store.

Критически важно установить zSupport до первого OTA-обновления. После обновления до версии 6.5+ без предустановленного zSupport установка через ADB или магазины невозможна — приложение просто не поставится. 👥 CONSENSUS

zSupport позволяет: переключать USB-порт между режимами Host (флешка) / Peripheral (ADB) / Auto без компьютера; входить в инженерное меню одной кнопкой; переключать язык интерфейса на русский без полной русификации (покрывает ~95% интерфейса, кроме приборки); автоматически выдавать права приложениям.

Обновления прошивки и судьба ADB

Обновление прошивки — самый болезненный момент для владельцев с открытым ADB. Существует два типа обновлений: малое OTA (обычно не закрывает ADB) и заводской сброс / major update (закрывает ADB, удаляет приложения, возвращает заводские настройки).

Обновление по воздуху (OTA) возможно бесплатно через китайский DNS. Устанавливается DNS Changer (com.andevstudioth.changedns с apkpure.net), выдаются VPN-права через ADB, прописываются DNS-серверы: 202.98.192.67, 188.166.226.78, 202.46.34.75 или 186.166.226.67. Далее вызывается меню обновления. 👥 CONSENSUS

Рекомендации перед обновлением: установить zSupport; держать порт в режиме Peripheral; сохранить APK инженерного меню для переустановки после обновления; на машинах 2024 года с zSupport обновление на 6.7 проходило без потери ADB.

Обновление по проводу (ZDS) стоит 40–60 тыс. руб. у специалистов. Считается безопаснее, но есть случаи зависания на определённом проценте, блокировки блоков и циклических перезагрузок — ремонт может стоить от 150 тыс. руб.

Русификаторы и ADB

Ситуация с русификаторами и ADB — одна из самых запутанных тем. Разные поставщики по-разному обходятся с доступом к инженерному меню.

Оставляют ADB открытым: Линко (Linko), Дейзи, лаунчер от Кирилла (10 000 руб. с самостоятельной установкой, продолжает работать после OTA-обновлений). 👥 CONSENSUS

Закрывают ADB (по отзывам владельцев): Xanavi, Можарова — блокируют доступ к инженерке для защиты интеллектуальной собственности. При этом Xanavi заявляет, что не закрывает ADB — остаётся только заводской динамический пароль. 🔥 DEBATED

Рекомендация: уточнять у установщика до русификации, останется ли ADB открытым, и ставить zSupport как страховку.

Спорные моменты

Закрывается ли ADB после OTA-обновления на 6.1+ / 6.7?

🔥 ОБСУЖДАЕТСЯ

Лагерь A: ADB остаётся, если установлен zSupport или машина 2024 года.
Владельцы машин 2024 года сообщают, что после обновления на 6.7 всё переключалось через zSupport. Если порт был в режиме Peripheral — доступ сохраняется. Можно сохранить APK инженерки до обновления и переустановить после.

Лагерь B: ADB закрывается паролем, особенно после сброса настроек.
После factory reset ADB закрывается. На 6.1.2+ самостоятельное открытие недоступно — требуется платная услуга. Без zSupport или с портом в Host-режиме ADB закроется паролем.

Нужен ли zSupport или достаточно стандартного ADB?

🔥 ОБСУЖДАЕТСЯ

Лагерь A: zSupport необходим перед обновлениями.
Множественные отчёты о потере ADB после OTA без переключалки. После версии 6.5 без предустановленного zSupport установка невозможна.

Лагерь B: Можно обойтись без zSupport.
Некоторые владельцы успешно обновлялись без потери ADB, если изначально был открыт через Info Hub или дилеров.

Безопасно ли использовать ADB App Control?

🔥 ОБСУЖДАЕТСЯ

Лагерь A: Удобный GUI-инструмент для быстрого управления пакетами.
Некоторые владельцы рекомендуют для блокировки системных приложений типа EVA.

Лагерь B: Опасно — может удалить нужные системные пакеты.
Опытные пользователи категорически против: приложение грохает нужные пакеты, система умирает, приходится восстанавливать. Лучше использовать команды ADB вручную.

Обновление по воздуху vs по проводу

🔥 ОБСУЖДАЕТСЯ

Лагерь A: OTA бесплатно и штатно — достаточно настроить китайский DNS.
Многие успешно обновлялись сами. Специалисты берут деньги за простую операцию.

Лагерь B: Самостоятельное обновление рискованно.
OTA может привести к циклическому ребуту, зависанию, закрытию инженерки. Обновление по проводу (ZDS) безопаснее, но при проблемах ремонт стоит от 150 тыс. руб.

Блокировка ADB русификаторами

🔥 ОБСУЖДАЕТСЯ

Лагерь A: Блокировка оправдана для защиты интеллектуальной собственности.
Разработчики вкладывают ресурсы; блокировка предотвращает кражу решений и проблемы с поддержкой.

Лагерь B: Блокировка вредит пользователю.
Невозможность самостоятельно устанавливать приложения, чинить баги, обновлять софт. Создаёт зависимость от установщика.

Предупреждения

  • ⚠️ Factory reset закрывает ADB и инженерку. Без сохранённого пароля восстановить доступ самостоятельно невозможно — придётся платить 10–60 тыс. руб. повторно.
  • ⚠️ zSupport не ставится после 6.5, если не был установлен ранее. Критически важно поставить до первого OTA.
  • ⚠️ Некоторые русификаторы блокируют инженерное меню. Уточняйте у установщика до русификации.
  • ⚠️ Верификатор пакетов сбрасывается после перезагрузки. Команды pm disable и settings put global нужно выполнять повторно. На 6.1+ не работают без открытой инженерки.
  • ⚠️ Root на рестайле полностью закрыт (6.0.2+). Попытка получить root может окирпичить головной блок. Magisk/su невозможны.
  • ⚠️ Удаление системных APK через ADB App Control может привести к вечному ребуту с закрытым ADB. Используйте команды ADB вручную.
  • ⚠️ Обновление прошивки стирает русификацию и настройки. Major update закрывает ADB, удаляет приложения, возвращает заводские настройки. Сохраняйте APK инженерки заранее.
  • ⚠️ Type-C → Type-C кабели часто не работают для ADB. Нужен Type-A → Type-C с поддержкой передачи данных.
  • ⚠️ MacBook на M-чипах может не видеть машину по ADB. Помогает другой компьютер или смена USB-порта.
  • ⚠️ Флешка не определяется, если порт в режиме Peripheral (ADB). Для флешки нужен режим Host. Только FAT32, exFAT не поддерживается.
  • ⚠️ Разрешения VPN для DNS Changer слетают после перезагрузки — нужно выдавать заново через ADB.
  • ⚠️ Обновление по проводу (ZDS) может привести к блокировке блоков. Ремонт — от 150 тыс. руб.
